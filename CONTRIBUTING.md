谢谢愿意贡献。在提 PR 之前请通读本页。

## 提 PR 之前

- 跑通本地测试 / lint(若有 CI 失败本项目不会合)
- commit message 用人话(中英文均可)、聚焦"为什么改",不要堆 "fix bug"
- 一个 PR 聚焦一件事,改动量保持 reviewer 友好
- 不要在 commit message / PR 里加 `Co-Authored-By: <bot or AI>` trailer

## 报 issue / 安全漏洞

- 普通 bug / feature → 直接开 issue
- **安全漏洞 / 隐私问题 → 不要公开提 issue**,先邮件联系作者(GitHub profile 公开邮箱),给作者修复 + 通报窗口
